## 2024-05-01 - Fix SSRF Vulnerability in Remote Sync
**Vulnerability:** Server-Side Request Forgery (SSRF) in the Batch AI Remote Sync feature. The `make_raw_curl_request` method is used in multiple AJAX handlers (e.g., `ajax_import_from_url`, `ajax_connect_remote_drafts`) to fetch data from URLs provided by the user (`$_POST["zip_url"]`, `$_POST["remote_url"]`). It uses raw `curl_init` instead of `wp_safe_remote_get`, bypassing the WordPress internal SSRF protections and allowing an authenticated attacker (manage_options) to make the server perform HTTP requests to arbitrary internal and external URLs, potentially exposing internal services or cloud metadata.
**Learning:** Using raw cURL ignores the built-in HTTP API protections in WordPress (like `wp_safe_remote_get()`), which handles loopback and local IP rejection. Even for administrative features, robust SSRF protection is necessary.
**Prevention:** Always use WordPress HTTP API functions (e.g., `wp_safe_remote_get`, `wp_safe_remote_request`) for outbound requests instead of raw cURL. If raw cURL must be used, implement strict URL and IP validation to block loopback, local, and cloud metadata IPs.
