## 2024-05-07 - Addressed Server-Side Request Forgery (SSRF) in Remote Sync
**Vulnerability:** The Batch AI remote sync process used raw `curl_init()` and `curl_exec()` to fetch remote data. This is a Server-Side Request Forgery (SSRF) vulnerability in WordPress because `curl` directly executes requests to arbitrary URLs provided in the database (or through input), bypassing the WordPress HTTP API. This allows malicious actors with `manage_options` capability (or if options are poisoned) to make the server perform requests to internal networks, loopback addresses, or arbitrary remote endpoints.
**Learning:** Raw `curl` calls in WordPress plugins bypass security features and proxy configurations built into the `WP_Http` class and specifically `wp_safe_remote_get`.
**Prevention:** Always use WordPress core HTTP functions like `wp_safe_remote_get()` when making outbound requests, as it enforces timeouts, handles proxy routing, and provides critical SSRF protections such as blocking requests to `localhost`, `127.0.0.1`, and internal IP ranges.
