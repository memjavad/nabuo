## 2024-05-03 - SSRF via curl_init() instead of wp_safe_remote_get()
**Vulnerability:** The `make_raw_curl_request` method in `includes/admin/batch-ai/class-batch-ai-remote-sync.php` uses `curl_init()` directly and disables SSL verification (`CURLOPT_SSL_VERIFYPEER = false`). It also does not protect against Server-Side Request Forgery (SSRF) since it does not restrict the destination IP address or use `wp_safe_remote_get()`.
